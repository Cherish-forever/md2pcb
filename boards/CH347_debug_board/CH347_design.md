# CH347F 多功能调试小板 — 设计文档

> 版本：v0.3（重新规划布局，精简扩展接口，0402封装，加拨码开关）
> 日期：2026-05-30
> 状态：草稿，待审核完善

## 1. 项目概述

基于 WCH CH347F（QFN28）设计一块 USB 高速转接调试小板，目标用途：

- USB Type-C 接口连接电脑
- 两侧 2.54mm 排针引出全部可用资源（朝上，可对插扩展板）
- STDC14 JTAG/SWD 调试座子（与 STLINK-V3MINI 接口兼容）
- 扩展板用于烧录 SPI Flash、I2C EEPROM 等存储器

## 2. 芯片选型

| 特性 | CH347F (QFN28) | CH347T (TSSOP20) |
|------|---------------|-------------------|
| 封装 | QFN28, 4×4mm | TSSOP20, 4.4×6.5mm |
| 工作模式 | 五合一，无需配置 | 需上电配置，四选一 |
| JTAG + SPI + I2C 同时 | 支持 | 不支持 |
| VIO 独立供电 | 支持 (1.2V~3.3V) | 不支持 |
| GPIO 数量 | 最多 8 路 | 模式相关 |
| 焊接难度 | 较高（QFN） | 较低（TSSOP） |

**结论：选择 CH347F。** 只有它能在同一块板上同时提供 JTAG、SPI、I2C、双串口，且支持 VIO 独立供电以匹配不同目标电压。

## 3. 板级布局

```
              ┌──────────────────────────┐
              │     USB Type-C (上边)     │
              │  (Sink, CC 5.1kΩ下拉)    │
              ├──────────────────────────┤
              │                          │
  左侧排针    │   ┌──────────────────┐   │   右侧排针
  J1 (10P)    │   │   CH347F QFN28   │   │   J2 (10P)
  SPI + I2C   │   │   (中心位置)     │   │   UART1完整
   + 电源     │   │                  │   │   + 电源
              │   │  [拨码开关]      │   │
              │   │  8MHz XTAL      │   │
              │   │  退耦电容       │   │
              │   └──────────────────┘   │
              │                          │
              ├──────────────────────────┤
              │   STDC14 座 (下边)       │
              │   JTAG/SWD + VCP 串口    │
              └──────────────────────────┘
```

- **上方**：USB Type-C 母座，VBUS 经 LDO 为板子供电
- **下方**：STDC14 1.27mm 2×7 座子（JTAG/SWD + 虚拟串口），与 STLINK-V3MINI 兼容
- **左侧**：2.54mm 排针 J1（10针），引出 SPI + I2C + 电源
- **右侧**：2.54mm 排针 J2（10针），引出 UART1 完整串口 + 电源
- 预估 PCB 尺寸：25×35mm，双面板
- **中心**：CH347F（QFN28），周围布置晶振、退耦电容、拨码开关
- **拨码开关**：2 位或 4 位，用于 VIO 选择、功能切换
- 两侧排针朝上焊接，可直接对插扩展板

## 5. 两侧排针定义

> 设计原则：最精简——5V / 3.3V / GND + SPI(4线) + I2C(2线) + UART1 完整串口。

### 5.1 左侧排针：SPI + I2C + 电源（J1，10针）

| 针号 | 信号名 | CH347F 脚 | 类型 | 备注 |
|------|--------|-----------|------|------|
| 1 | 5V | — | PWR | USB VBUS 直出 |
| 2 | 3V3 | — | PWR | 板载 LDO 输出 |
| 3 | GND | — | PWR | |
| 4 | SPI_SCK | 14 | O | SPI 时钟 |
| 5 | SPI_MOSI | 16 | O | SPI 数据输出 |
| 6 | SPI_MISO | 15 | I (FT) | SPI 数据输入 |
| 7 | SPI_CS | 13 | O | SPI 片选 |
| 8 | I2C_SCL | 11 | O | I2C 时钟（⚠️ 复用 CTS1） |
| 9 | I2C_SDA | 12 | I/O (FT) | I2C 数据（⚠️ 复用 RTS1） |
| 10 | GND | — | PWR | |

### 5.2 右侧排针：UART1 完整串口 + 电源（J2，10针）

| 针号 | 信号名 | CH347F 脚 | 类型 | 备注 |
|------|--------|-----------|------|------|
| 1 | 5V | — | PWR | USB VBUS 直出 |
| 2 | 3V3 | — | PWR | 板载 LDO 输出 |
| 3 | GND | — | PWR | |
| 4 | UART1_TX | 4 (TXD1) | O | 串口1发送 |
| 5 | UART1_RX | 5 (RXD1) | I | 串口1接收 |
| 6 | UART1_RTS | 12 (RTS1) | O | 串口1 RTS（⚠️ 复用 I2C_SDA） |
| 7 | UART1_CTS | 11 (CTS1) | I (FT) | 串口1 CTS（⚠️ 复用 I2C_SCL） |
| 8 | UART1_DTR | 7 (DTR1) | O | 串口1 DTR / TNOW1（RS485方向控制） |
| 9 | GND | — | PWR | |
| 10 | GND | — | PWR | |

> **说明**：
> - UART1 引出完整 MODEM 信号（TX/RX/RTS/CTS/DTR），可直接对接 RS232/RS485 扩展板
> - UART1_DTR 可通过拨码 Bit2 切换为 TNOW1，用于 RS485 自动方向控制
> - **I2C（J1 Pin 8/9）与 UART1 流控（J2 Pin 6/7）复用 Pin 11/12，同时只能用一个**
>   - I2C 烧录存储器时：UART1 的 RTS/CTS 不可用，TX/RX/DPR 仍正常
>   - RS232/RS485 扩展板时：I2C 不可用，SPI 仍正常
> - 所有 I/O 电平由 VIO 电压决定（默认板载 3.3V，可由拨码 Bit1 切换到 STDC14 输入）

## 6. STDC14 座子（J3）引脚定义

与 STLINK-V3MINI (CN5) 完全兼容，14 针 1.27mm 间距 2×7 排针座。核心 10 针遵循 ARM Cortex 调试标准（MIPI10/ARM10）。

| Pin | 信号 | 方向 | CH347F 脚 | 说明 |
|-----|------|------|-----------|------|
| 1 | T_VCP_RX | 调试器→目标 | TXD0 (19) | 目标板 UART 接收 |
| 2 | T_VCP_TX | 目标→调试器 | RXD0 (22) | 目标板 UART 发送 |
| 3 | VCC / VTref | 目标→调试器 | VIO (6) | 目标 3.3V 参考电压 |
| 4 | JTMS / SWDIO | 调试器→目标 | TMS (26) | JTAG TMS / SWD 数据 |
| 5 | GND | — | GND | 地 |
| 6 | JTCK / SWCLK | 调试器→目标 | TCK (23) | JTAG 时钟 / SWD 时钟 |
| 7 | GND | — | GND | 地 |
| 8 | JTDO | 目标→调试器 | TDO (24) | JTAG TDO |
| 9 | NC | — | — | ARM10 KEY 位置，悬空 |
| 10 | JTDI | 调试器→目标 | TDI (25) | JTAG TDI |
| 11 | GNDDetect | 目标→调试器 | — | 目标端接地以检测连接 |
| 12 | NRST | 调试器→目标 | SRST (8) | 目标系统复位 |
| 13 | TRST | 调试器→目标 | TRST (9) | JTAG TAP 复位（可选） |
| 14 | NC | — | — | 预留（可接 GPIO） |

> **注意**：Pin 3 的 VCC 来自目标板，通过 STDC14 排线接到 CH347F 的 VIO 引脚，实现电平自动匹配。如果目标板不提供 VCC，需要在 J1 的 Pin 12 或 VIO 跳线上手动连接 3.3V。

### 6.1 STDC14 串口指示灯

在 STDC14 座子附近放置两个 LED，用于指示 UART0（T_VCP）的收发状态。

**TX 指示灯**（调试器→目标方向）：
```
3V3 ── LED_TX ── R_TX (1kΩ) ── TXD0 (Pin 19)
```
- 空闲时 TXD0 为高电平，LED 灭
- 发送数据时 TXD0 拉低，LED 闪亮
- TXD0 输出驱动能力足够（推挽），可直接驱动 LED

**RX 指示灯**（目标→调试器方向）：
```
3V3 ── LED_RX ── R_RX (1kΩ) ── RXD0 (Pin 22)
```
- 空闲时 RXD0 被目标拉高（或 CH347 内置上拉维持高电平），LED 灭
- 接收数据时 RXD0 拉低，LED 闪亮
- RXD0 为输入引脚，内置约 40kΩ 上拉。LED 电流约 (3.3V - 1.8V) / 1kΩ ≈ 1.5mA，不影响信号质量

> **布局建议**：两颗 LED 放在 STDC14 座子同侧，丝印标注 TX/RX 或 ←/→ 方向。建议用不同颜色区分（如 TX 绿色、RX 黄色，或同色均可）。电源 LED 用红色。

### 6.2 信号保护

STDC14 的 JTAG/SWD 信号线（Pin 4/6/8/10/12）建议串联 22Ω～100Ω 小电阻，用于：
- 抑制振铃和反射
- 限制短路电流
- ESD 保护配合

所有 STDC14 数字 I/O 线建议对地加 TVS 管（如 SRV05-4 阵列），保护 CH347F 引脚。

### 6.3 目标供电开关

通过一个 3 段滑动开关（SP3T），选择是否通过 STDC14 给目标板供电，以及供电电压。

```
                      ┌─── 断开（默认）
5V ──────────────────┤
                      ├─── 3.3V ──┬── STDC14 Pin 13 (T_PWR)
板载 3.3V ───────────┤           │
                      └─── 5V ────┘
```

| 开关位置 | 功能 | 目标供电 | VIO 参考来源 |
|----------|------|----------|-------------|
| 断开 | 目标自供电 | 无 | 目标板→STDC14 Pin 3→VIO |
| 3.3V | 调试器供 3.3V | Pin 13 输出 3.3V | Pin 3 同时做 VIO 参考（与 Pin 13 同源） |
| 5V | 调试器供 5V | Pin 13 输出 5V | 目标板需自备 LDO 降压后回传 Pin 3 做 VIO 参考 |

> **注意**：
> - CH347F 的 VIO 引脚最高耐受 3.6V，所以 VIO 永远只接 Pin 3，不接 Pin 13 的 5V 输出
> - 5V 档仅当目标板有板载 3.3V LDO、且将 3.3V 回传到 Pin 3 时才安全
> - Pin 13 在标准 STDC14 中为 NC，此处重新定义为 T_PWR（目标供电输出）
> - 目标板上 Pin 13 应悬空（不使用此引脚的板子不受影响）

## 7. 电路原理

### 7.1 USB Type-C 接口

```
USB Type-C 母座
├── VBUS ──── PTC (500mA) ──┬── LDO 输入端
│                           └── 10µF + 0.1µF 滤波
├── D+  ──────── CH347F Pin 28 (UD+)  (不可串接电阻！)
├── D-  ──────── CH347F Pin 27 (UD-)  (不可串接电阻！)
├── CC1 ──── 5.1kΩ ──── GND
├── CC2 ──── 5.1kΩ ──── GND
├── GND ──────── 公共地
└── SHIELD ───── 外壳接地（可选经 1MΩ + 4.7nF 并联）
```

> **重要**：CH347 的 UD+/UD- 引脚**绝对不能串接电阻**（手册 4.1 节明确警告），USB 收发器已内置匹配。
> **PTC 自恢复保险**：500mA（如 MF-NSMF050，SMD 1206），防止扩展板短路拉垮 USB 端口。

### 7.2 电源

```
VBUS (5V) ── LDO (3.3V) ── VCC (Pin 21)
              │
              ├── 10µF 输入电容
              ├── 0.1µF 输入退耦
              ├── 10µF 输出电容
              └── 0.1µF 输出退耦

VIO (Pin 6) ── 拨码开关 Bit1 选择:
                ├── OFF（默认）：板载 3.3V ──┬── VIO
                │                          │
                └── ON：STDC14 Pin 3 ───────┘
                          ↑
                    目标板 Vref（1.8~3.3V）

> VIO 是 CH347F 的 I/O 端口独立电源引脚，决定所有数字 I/O（SPI/I2C/UART/JTAG）的信号电平。VCC（Pin 21）固定 3.3V 为核心供电，VIO 可独立于 VCC 设置为 1.8V/2.5V/3.3V，从而直接对接不同电压的目标器件。CH347T 没有 VIO，所有 I/O 固定 3.3V，这也是本设计选 CH347F 的关键原因之一。
```

**防反灌**: 当 VIO 由 STDC14 Pin 3（目标板参考电压）提供时，如果目标板电压高于板载 LDO 的 3.3V，会通过拨码开关倒灌到 LDO 输出端。在板载 3.3V 到拨码开关之间串联一个肖特基二极管（如 BAT54C，Vf≈0.2V），阻断反向电流。

```
板载 3.3V ── BAT54C ── 拨码 Bit1 ── VIO (Pin 6)
              (防反灌)
```

推荐 LDO：ME6211C33M5G（SOT-23-5，2.9×1.6mm），压差 100mV@100mA，无需散热，适合 60mA 以内负载。备选 XC6206P332MR（SOT-23，同样小巧）。

### 7.3 时钟

```
CH347F Pin 1 (XI) ──── 8MHz 晶振 ──── CH347F Pin 2 (XO)
                       │         │
                     22pF      22pF
                       │         │
                      GND       GND
```

晶振频率 8MHz ±0.4%，CL = 20pF 左右。可选并联 1MΩ 电阻帮助起振（非必需）。

### 7.4 复位

```
CH347F Pin 3 (RST#) ──── 10kΩ 上拉到 VIO
                      └──── 可选按键到 GND（低电平复位）
```

内置上拉电阻约 40kΩ。外部 10kΩ 上拉增强抗干扰，加一个轻触开关便于手动复位。

### 7.5 退耦电容

每个电源引脚（VCC Pin 21、VIO Pin 6）就近放置 0.1µF 退耦电容。

### 7.6 拨码开关

使用 4 位 DIP 拨码开关（1.27mm 或 2.54mm 间距），各开关定义：

| 位 | 功能 | ON | OFF（默认） | 连接 |
|----|------|-----|------------|------|
| 1 | VIO 来源 | 外部（STDC14 Pin 3） | 板载 3.3V | CH347F Pin 6 (VIO) |
| 2 | TNOW 使能 | DTR0→TNOW0, DTR1→TNOW1 | 正常 DTR 功能 | DTR0 (Pin 10) 经 4.7kΩ 下拉 |
| 3 | 默认参数 | 禁用内部 EEPROM，使用芯片默认参数 | 使用 EEPROM 配置 | RTS0 (Pin 18) 经 4.7kΩ 下拉 |
| 4 | 预留 | — | — | 引出到测试点 |

**原理**：
- Bit 1：VIO 通过拨码开关选择板载 3.3V 或外部参考电压，支持 1.8V/2.5V/3.3V 目标电平
- Bit 2：闭合后 DTR0 引脚检测到下拉，驱动将 DTR0/DTR1 切换为 TNOW0/TNOW1（RS485 方向控制）
- Bit 3：闭合后 RTS0 引脚检测到下拉，芯片忽略内部 EEPROM 配置，使用出厂默认 VID/PID/参数

> 替代方案：Bit 2 和 Bit 3 也可以用 0Ω 跳线电阻代替拨码开关（节省空间和成本），因为这两个功能一般不需要频繁切换。

## 9. 扩展板设计思路

扩展板通过标准 2.54mm 排针母座（10×2）对插在核心板上方。

### 9.1 SPI Flash 烧录扩展板

```
J1 对应排母 ──┬── 3V3     → SOP8 Pin 8 (VCC)
              ├── GND      → SOP8 Pin 4 (GND)
              ├── SPI_SCK  → SOP8 Pin 6 (CLK)
              ├── SPI_MOSI → SOP8 Pin 5 (DI)
              ├── SPI_MISO → SOP8 Pin 2 (DO)
              └── SPI_CS0  → SOP8 Pin 1 (CS#)

板上器件：
- 1× SOP8 宽体测试座（通用 25XX 系列）
- 可选 SOP16/DIP8 兼容焊盘
```

### 9.2 I2C EEPROM 烧录扩展板

```
J1 对应排母 ──┬── 3V3      → SOP8 Pin 8 (VCC)
              ├── GND       → SOP8 Pin 4 (GND) + A0/A1/A2
              ├── I2C_SCL   → SOP8 Pin 6 (SCL)
              └── I2C_SDA   → SOP8 Pin 5 (SDA)

板上器件：
- 1× SOP8 测试座（24CXX 系列）
- 可选 I2C 上拉电阻（CH347 内置上拉，一般不需要）
```

### 9.3 通用转接扩展板

将 STDC14 信号转接到其他接口标准：
- 20 针 ARM JTAG (2×10, 2.54mm)
- Tag-Connect TC2050（SWD 免焊线缆）
- 2.54mm 单排针（杜邦线直接连接）

## 10. 待确认事项

以下设计点需要在后续迭代中确认：

- [x] **排针数量**：10针×2，SPI + I2C + UART1 完整串口 + 电源
- [ ] **板子最终尺寸**：当前暂定 25×35mm，需实际布局后确认
- [ ] **VIO 供电策略**：是否需要板载 1.8V/2.5V 可调 LDO，还是完全依赖外部参考
- [ ] **STDC14 座子品牌**：SAMTEC FTSH-107 或替代品（如中国产兼容型号）
- [x] **LED 指示灯**：电源（红色）+ STDC14 串口 TX（绿色）/ RX（黄色）
- [x] **防反插/防短路**：已加入 JTAG 阻尼电阻 + TVS 保护方案，USB ESD 保护
- [ ] **VIO 反灌保护**：当 STDC14 Pin 3 提供 VIO 时，需要防止电流倒灌到板载 3.3V LDO
- [ ] **PCB 工艺**：2 层板 vs 4 层板，1.0mm vs 1.6mm 厚度
- [ ] **生产数量**：手焊 vs 钢网+SMT 打样，影响封装选择
