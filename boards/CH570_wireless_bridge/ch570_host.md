# CH570 无线串口桥 — 主机端设计文档

> 版本：v0.1（初稿）
> 日期：2026-06-08
> 状态：草稿，待审核完善
> 配套文档：`ch570_device.md`（从机端 / 目标端）

## 1. 角色定位

**主机端**：插在 PC 的 USB 口上，PC 识别为虚拟串口（COM 口）。主机端通过 2.4G 无线与从机端通信，实现 PC ↔ 目标 MCU 的无线串口桥接。

```
PC 串口终端 ──USB(CDC)──► CH570 主机 ──2.4G──► CH570 从机 ──UART──► 目标 MCU
```

- **上行（PC→目标）**：PC 串口数据 → USB CDC 接收 → CH570 缓冲 → 2.4G 发送
- **下行（目标→PC）**：2.4G 接收 → CH570 缓冲 → USB CDC 发送 → PC 串口终端

## 2. 芯片选型

| 特性 | CH570（主机端要求） |
|------|-------------------|
| USB Device 模式 | ✅ 必须（虚拟串口） |
| 2.4G 无线 | ✅ 必须 |
| UART | 不需要（目标不在这一侧） |
| GPIO | 少量（状态 LED） |
| 推荐封装 | **SOP8** 或 **TSSOP16**（手工焊接友好） |

> 主机端不需要 UART。USB + 2.4G 两个外设就够。封装选 SOP8（8 脚）最精简，连 LED 都省掉都可以。

## 3. 硬件极简设计

### 3.1 最小系统（SOP8）

```
          ┌──────○──────┐
    GND ──┤1          8├── VCC (5V 或 3.3V)
     NC ──┤2   CH570  7├── 2.4G RF 天线（PCB 天线 / 陶瓷天线）
     NC ──┤3  (SOP8)  6├── USB D- (直连)
     NC ──┤4          5├── USB D+ (直连)
          └─────────────┘
```

> SOP8 封装的 CH570 只剩 6 个可用引脚（除去电源）。USB D+/D- 占 2 个，2.4G RF 占 1 个，剩下 3 个可做状态 LED 或按键。

### 3.2 电路要点

**USB 接口**（可选 Type-C 或 Micro USB）：

```
USB 母座 (Type-C 2.0)
├── VBUS ──→ CH570 VCC (5V，芯片内置 LDO)
├── D+  ──→ CH570 USB D+ (直连，不需外部电阻)
├── D-  ──→ CH570 USB D- (直连)
└── GND ──→ 公共地
```

> CH570 内置 5V→3.3V LDO 和 USB 收发器终端电阻，**USB D+/D- 不需要外接任何电阻**。跟 CH347 一样的设计哲学。

**2.4G 天线**：

```
CH570 RF 引脚 ──┬── 电感匹配网络（如果需要）
                └── PCB 天线 或 陶瓷天线（如 AN2051）
```

WCH 官方 EVT 开发板通常提供参考天线设计，直接抄。

**状态 LED（可选）**：

```
CH570 GPIO ── 1kΩ ── LED ── GND
```

- 上电常亮 = 等待对端连接
- 慢闪 = 已连接，空闲
- 快闪 = 数据传输中

### 3.3 供电

```
USB VBUS (5V) ──→ CH570 VCC
                      │
                 内置 LDO → 3.3V（内核 + 2.4G 射频）
```

- 无需外置 LDO，CH570 内置 5V→3.3V 稳压器
- USB VBUS 直连芯片 VCC 引脚
- 建议 VCC 就近放 0.1µF + 10µF 退耦电容

## 4. 固件架构

### 4.1 数据流

```
          ┌──────────────────────────────────┐
          │         CH570 主机固件             │
          │                                  │
USB CDC ──►│  RX 环形缓冲 (1KB)              │
          │        ↓                        │
          │  2.4G 发送队列                   │──► 2.4G TX
          │                                  │
          │  2.4G 接收队列                   │
          │        ↓                        │
          │  TX 环形缓冲 (1KB)               │──► USB CDC
          │                                  │
          └──────────────────────────────────┘
```

### 4.2 固件模块

| 模块 | 功能 | 依赖 |
|------|------|------|
| USB CDC 驱动 | 枚举为虚拟串口，收发数据 | WCH 官方 USB 库 |
| 2.4G 协议栈 | 无线收发、自动重传、ACK | WCH 官方 RF 库 |
| 环形缓冲 | 双向 1KB 缓冲，解耦 USB 和 2.4G 速率 | 自写（<50 行） |
| 连接管理 | 配对、心跳、超时重连 | 自写 |
| LED 指示 | GPIO 控制状态灯 | 自写 |

### 4.3 USB CDC 描述符

```c
// PC 端识别为 "CH570 Wireless Bridge (Host)"
// VID: 0x1A86 (WCH), PID: 自定义
// 波特率无实际意义（无线传输不限波特率），
// 但建议上报 115200 或 921600 以兼容串口软件
```

### 4.4 2.4G 无线参数

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 速率 | 2Mbps | 最高速度，覆盖串口所有波特率 |
| 频道 | 可配置（2.400-2.483GHz） | 避开 WiFi 拥挤频道 |
| 发射功率 | +7dBm | 最大功率，室内 10-30 米 |
| 包大小 | 32 字节/包 | 平衡延迟和吞吐 |
| 重传 | 自动重传 + ACK | WCH RF 库内置 |
| 配对 | 固定地址 / 按键配对 | 防止串扰 |

### 4.5 关键固件逻辑（伪代码）

```
初始化:
    USB_CDC_Init()
    RF_Init(channel, tx_power)
    RF_SetAddress(host_addr, device_addr)
    LED_Init()

主循环:
    // 上行: PC → 无线
    if USB_CDC_RX_Available():
        data = USB_CDC_Read(up_to_32_bytes)
        RF_Send(data)              // 自动 ACK + 重传
        LED_Flash_TX()

    // 下行: 无线 → PC
    if RF_Data_Available():
        data = RF_Read()
        USB_CDC_Write(data)
        LED_Flash_RX()

    // 心跳
    if heartbeat_timeout:
        RF_Send_Beacon()           // 维持连接
```

## 5. PCB 设计考虑

| 参数 | 建议 |
|------|------|
| 尺寸 | **越小越好**，类似 U 盘大小（15×35mm） |
| 封装 | SOP8（手工焊）或 DFN10（最小体积） |
| 板厚 | 1.0mm（插 USB 口轻便） |
| 层数 | 2 层即可 |
| 天线 | PCB 蛇形天线或陶瓷天线 |
| USB 插头 | Type-C 公头直插（像 U 盘） 或 Type-C 母座 + 短线 |

## 7. 与从机端的协作

| 通信方向 | 主机端职责 | 从机端职责 |
|----------|-----------|-----------|
| PC→目标 | USB CDC 收 → 2.4G 发 | 2.4G 收 → UART 发 |
| 目标→PC | 2.4G 收 → USB CDC 发 | UART 收 → 2.4G 发 |
| 连接管理 | 发起配对，发送心跳 | 响应配对，回复心跳 |
| 速率匹配 | 2.4G 端 2Mbps | UART 端跟随目标 MCU 波特率 |
